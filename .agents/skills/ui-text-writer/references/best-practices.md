# UI Text Best Practices

## General Tone

- **Professional & Trustworthy**: Avoid slang or overly casual language.
- **Friendly & Human**: Use "We" and "You" to create a connection. Avoid robotic phrasing.
- **Concise**: Remove unnecessary words. Users scan, they don't read.

## Mechanics

### Capitalization

- **Headings/Titles**: Title Case (e.g., "Create New Account").
- **Buttons**: Title Case (e.g., "Save Changes").
- **Body Text**: Sentence case (e.g., "Please enter your email address.").
- **Labels**: Sentence case (e.g., "Email address").

### Punctuation

- **Headings/Labels**: No exclamation marks or periods.
- **Full Sentences**: Use periods.
- **Microcopy**: Avoid unnecessary punctuation in isolated phrases.

## Component Specifics

### Buttons

- **Action-Oriented**: Start with a strong verb (e.g., "Save", "Delete", "Continue").
- **Clear Consequence**: The user should know exactly what will happen.
- **Avoid Generic Labels**: Prefer "Save Changes" over "Ok" or "Yes".

### Error Messages

- **Explain the Problem**: What went wrong?
- **Propose a Solution**: How can the user fix it?
- **Avoid Blame**: Don't say "You entered invalid data". Say "The password is too short".

### Onboarding

- **Focus on Value**: Explain *why* the user should care, not just *how* it works.
- **One Idea Per Screen**: Don't overwhelm the user.

### Empty States

- **Delightful**: Use this space to educate or encourage.
- **Actionable**: Always provide a way to fill the empty state (e.g., "Create your first project").
